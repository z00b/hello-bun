# Distribution Setup

This document explains how to set up the various distribution channels for hellbun.

## Homebrew Tap

### Prerequisites

1. Create a new GitHub repository called `homebrew-tap` under your account/org
2. Create a GitHub Personal Access Token with `repo` scope
3. Set the token as a secret in your CI/CD or use it locally with GoReleaser

### Repository Setup

```bash
# Create the homebrew-tap repository
gh repo create z00b/homebrew-tap --public --description "Homebrew tap for z00b packages"
```

### Testing Locally

```bash
# After running goreleaser
brew tap z00b/tap https://github.com/z00b/homebrew-tap
brew install hellbun
```

### Usage for End Users

```bash
brew tap z00b/tap
brew install hellbun
```

## Nix User Repository (NUR)

### Automated Publishing

The CircleCI workflow automatically publishes to your NUR repository:
1. GoReleaser generates the Nix package with proper hashes
2. A separate step clones your `nur-packages` repository
3. Copies the generated package to the correct location
4. Adds a `flake.nix` file for Nix Flakes support (if not already present)
5. Commits and pushes to the NUR repository

This approach avoids `nix-prefetch-url` timing issues while still fully automating the release process.

### Nix Flakes Support

The repository includes a `flake.nix` that will be published to your NUR repository, enabling users to install via:

```bash
# Run directly
nix run github:z00b/nur-packages#hellbun

# Install to profile
nix profile install github:z00b/nur-packages#hellbun

# In a flake.nix
{
  inputs.z00b.url = "github:z00b/nur-packages";
  # ...
  packages = [ z00b.packages.${system}.hellbun ];
}
```

### Prerequisites

1. Fork the NUR repository: https://github.com/nix-community/NUR
2. Create a repository called `nur-packages` with your Nix package definitions
3. Register your repository in the NUR by submitting a PR to the NUR repo

### Repository Setup

```bash
# Create the nur-packages repository
gh repo create z00b/nur-packages --public --description "Nix User Repository packages"
```

### NUR Repository Structure

Your `nur-packages` repository should have this structure:

```
nur-packages/
├── default.nix          # Main entry point
├── pkgs/
│   └── hellbun/
│       └── default.nix  # Generated by GoReleaser
└── README.md
```

### Example `default.nix` for NUR

```nix
{ pkgs ? import <nixpkgs> {} }:

{
  lib = import ./lib { inherit pkgs; };
  modules = import ./modules;
  overlays = import ./overlays;

  # Package definitions
  hellbun = pkgs.callPackage ./pkgs/hellbun { };
}
```

### Registering with NUR

1. Add your repository to the NUR by creating a PR to https://github.com/nix-community/NUR
2. Edit `repos.json` to include:

```json
{
  "repos": {
    "z00b": {
      "url": "https://github.com/z00b/nur-packages"
    }
  }
}
```

### Usage for End Users

After your NUR is registered, users can install with:

```nix
# In configuration.nix or home.nix
{ pkgs, ... }:
let
  nur = import (builtins.fetchTarball "https://github.com/nix-community/NUR/archive/master.tar.gz") {
    inherit pkgs;
  };
in {
  environment.systemPackages = [
    nur.repos.z00b.hellbun
  ];
}
```

Or via command line:

```bash
nix-env -iA nur.repos.z00b.hellbun
```

## CI/CD Integration

### CircleCI Configuration

The project is configured to use CircleCI for automated releases (`.circleci/config.yml`).

#### Setup Steps:

1. **Connect your repository to CircleCI:**
   - Go to https://circleci.com/
   - Click "Set Up Project" for your repository
   - CircleCI will automatically detect the `.circleci/config.yml`

2. **Set up the `z00b-releaser` context in CircleCI:**
   - Go to Organization Settings → Contexts
   - Create or use the existing `z00b-releaser` context
   - Add `GITHUB_TOKEN` with a GitHub Personal Access Token (needs `repo` scope)
   - Optional: Add `HOMEBREW_TAP_TOKEN` if publishing to Homebrew tap
   - See `.circleci/README.md` for detailed instructions

3. **Enable tag builds:**
   - By default, CircleCI only builds branches
   - The workflow is configured to trigger only on version tags (`v*`)

#### GitHub Personal Access Token:

**See [GITHUB_TOKEN_PERMISSIONS.md](GITHUB_TOKEN_PERMISSIONS.md) for detailed permission requirements.**

Quick summary:
- **Fine-grained token (Recommended)**: Contents (Read and write) for all three repos
- **Classic token**: `repo` scope

#### Testing the workflow:

```bash
# Create and push a tag to trigger the release
git tag -a v1.0.0 -m "First release"
git push origin v1.0.0

# CircleCI will automatically:
# 1. Install Bun and GoReleaser
# 2. Build binaries for all platforms
# 3. Create GitHub release
# 4. Update Homebrew formula
# 5. Update Nix package
```

## Updating Releases

When you push a new tag, CircleCI will automatically:
1. Build binaries for all platforms
2. Create a GitHub release
3. Update the Homebrew formula in your tap
4. Generate the Nix package with proper hashes
5. Push the Nix package to your NUR repository

```bash
git tag -a v1.0.1 -m "Release v1.0.1"
git push origin v1.0.1
# GitHub Actions or local goreleaser will handle the rest
```
